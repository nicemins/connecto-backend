# 📱 Connecto - 5분 익명 랜덤 통화 서비스 PRD

## 1. 서비스 정의 (Core Concept)
- **정의**: "사람을 고르는 대신, 지금 바로 말할 수 있는 연결을 제공하는 서비스"
- **슬로건**: "지금, 누군가와 5분만 이야기해요."
- **핵심 철학**: 
  - 프로필 판단 배제 (목소리 우선)
  - 관계 형성 부담 최소화 (5분 후 자동 종료)
  - 감정 상처 방지 (거절 알림 없음)

## 2. 핵심 UX 흐름 (Happy Path)
1. **로그인**: 소셜 또는 이메일을 통한 최소 정보 가입.
2. **랜딩**: [지금 전화하기] 버튼이 강조된 심플한 메인 화면.
3. **매칭**: 버튼 클릭 시 대기열 진입 및 실시간 1:1 매칭.
4. **통화**: 
   - 상대 정보는 오직 '언어'와 '나이대'만 노출.
   - 최대 5분 타이머 작동.
   - 30초 이후부터 자유롭게 종료 가능.
5. **통화 종료 후 (Reveal)**:
   - 상대방의 프로필 카드(사진, 닉네임, 자기소개) 공개.
   - 10초 이내에 [👍 다시 통화] 또는 [👋 종료] 선택.
6. **재연결**: 양측 모두 👍를 누른 경우에만 다시 연결되거나 친구 관계 형성.

## 3. 백엔드 핵심 기능 및 API 명세 (MVP)
### 인증 (Auth)
- `POST /auth/login`: 최소 단계 로그인.
- `GET /users/me`: 내 상태 조회.

### 매칭 & 통화 (Match & Call)
- `POST /match/start`: 대기열 진입 (Redis 기반 FIFO 권장).
- `POST /match/cancel`: 대기열 이탈.
- `GET /match/status`: 매칭 상태 확인 및 WebRTC 방 정보 수신.
- `POST /call/end`: 통화 종료 기록.
- `POST /call/again`: 재연결(👍) 의사 표현.

### 프로필 (Profile)
- `GET /match/result/{sessionId}`: 통화 종료 후 상대방 프로필 조회 (권한 필수).

## 4. 제거 및 제외 항목 (MVP Out of Scope)
- **삭제 대상**: 게시글(Post), 클럽(Club), 관심사 태그(Interest) 상세 관리.
- **제외 기능**: 실시간 채팅, 친구 목록 검색, 사진 미리보기, 랭킹 시스템.

## 5. 기술적 가이드라인
- **Real-time**: WebRTC를 활용한 음성 통화 (Agora 등 SDK 활용 고려).
- **Concurrency**: 동시 접속자 매칭 시 교착 상태(Deadlock) 방지.
- **Security**: 통화 세션이 유효한 유저에게만 상대 프로필 정보를 일시적으로 노출.
